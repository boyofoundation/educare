### CRITICAL learnings (apply to Phase 2/3/4)
1. **isomorphic-git workdir walker does NOT skip `.git`.** statusMatrix returns `.git/index`, `.git/HEAD`, `.git/objects/...` as `[filepath,0,2,0]` (untracked). If staged → `.git` files get COMMITTED into the tree → next statusMatrix reads HEAD tree containing `.git` → isomorphic-git throws `UnsafeFilepathError: The filepath ".git" contains unsafe character sequences`. FIX: `isGitInternalPath(filepath)` (any segment === '.git') filter on every statusMatrix row BEFORE git.add/remove (in commitAll born path, status(), diff() workdir collect). Note: `.educare` must NOT be filtered here (it's committed metadata).
2. **`git.add` needs `force: true`** to override `.gitignore` (D4 "ignored" semantics). statusMatrix(`ignored:true`) only makes ignored files VISIBLE; add still respects .gitignore without force.
3. **`.educare/meta.json` IS committed** (travels with tree for metadata versioning). Filtered only from USER-FACING lists: log().files (via listCommitFiles isReservedPath), status() results, diff() results. restoreCommitTree restores it (walk doesn't filter). commitAll unborn+born paths stage it.
4. **commit message trailing newlines**: isomorphic-git stores `'body\n\nTrailer: v\n'`. stripTrailers must `replace(/\n+$/,'')` before splitting, else trailing empty line breaks the backward trailer scan.
5. **`isSnapshot:true` ⇒ `allowEmpty:true`** (snapshots always record state = createSnapshot semantics). `allowEmpty = options.allowEmpty ?? Boolean(isSnapshot)`. run-start dedup (D4) is the CALLER's (store) responsibility before calling.
6. **WalkerEntry.content() returns `Promise<Uint8Array | void>`** — use `toBytes()` helper (void → empty Uint8Array) for Uint8Array constructor type safety.
7. **readCommit result**: tree oid is at `result.commit.tree` (not `.tree`).
8. **statusMatrix column semantics confirmed**: [filepath, HEAD(0|1), WORKDIR(0|1|2), STAGE(0|1|2|3)]. WORKDIR=2 detects modification. Recipe: workdir===0&&head===1→git.remove; workdir===2→git.add(force).
9. **diff 5.x ships NO types; @types/diff@8 is a deprecated stub resolving nothing.** Wrote ambient `diff.d.ts` shim (createPatch/diffLines/diffWords/diffChars + Change/Options types) mirroring csstree-validator.d.ts. lightning-fs 4.6.2 HAS built-in index.d.ts (no shim needed).

### CRITICAL learning — isomorphic-git stat-cache false-unchanged (would cause SILENT DATA LOSS)
isomorphic-git `statusMatrix`/workdir walker uses a **stat-cache fast-path** (`compareStats`): it reuses the cached index SHA when the file's `size`, `mtimeSeconds`, `ctimeSeconds`, and `ino` all match the staged entry — WITHOUT recomputing the content SHA. For files edited to the **same byte size within the same second** as the previous `git.add` (e.g. `v2`→`v3`, or any single-char edit committed fast), statusMatrix reports `unchanged` and the edit is DROPPED from the commit. LightningFS mtime is second-granular and inode is stable per path, so the cache hits. This is unacceptable for local versioning.
**FIX**: `commitAll` and `status` NO LONGER use statusMatrix. They detect changes **content-based**: walk HEAD tree → `path→blobOid` (via `entry.oid()`, reads committed objects reliably); for each workdir file compute `git.hashBlob({object: bytes})` and compare. Deletions = HEAD files absent from workdir. This is correct regardless of stat cache. `diff()` was already content-based (bytesEqual) — unaffected.
Spike `gitStatus`/`commitAll` tests pass with the rewrite.

### CRITICAL FIX — browser Buffer polyfill (reported by user: "GIT似乎無法COMMIT" + readFile errors)
**Root cause**: isomorphic-git internally uses Node's `Buffer` (`_GitIndex.from` / `updateCachedIndexFile` during git.add/commit/index ops). Browsers have NO `Buffer` global → `ReferenceError: Buffer is not defined` on ANY commit path (gitCommit, createSnapshot, run-start snapshot, migration). This crashed the agent turn. readFile/writeFiles (LightningFS-only, no isomorphic-git) were NOT affected — the user's "readFile errors" were the cascade from the crashed turn. Tests passed because vitest/Node provide Buffer.
**Fix**: `ensureBufferPolyfill()` in htmlProjectGitService — lazily `import('buffer')` and set `globalThis.Buffer` before the dynamic `import('isomorphic-git')` (and lightning-fs). Added `buffer@6.0.3` dep. Lazy (D7 — only loads when git used).
**Verification**: chrome-devtools browser eval of the REAL tool path (executeHtmlProjectToolCall) — writeFiles/readFile/createSnapshot/gitCommit/gitLog/gitStatus ALL pass in the browser after the polyfill.
**Regression guard**: cannot test in vitest (Node needs Buffer; deleting globalThis.Buffer destroys the process — confirmed 752k errors). Browser-verified instead; documented in spike test file.
**Lesson**: dynamic-imported Node-origin libs (isomorphic-git) need browser polyfills (Buffer, possibly others). Verify in REAL browser, not just jsdom — jsdom runs under Node so hides Node-global dependencies.
